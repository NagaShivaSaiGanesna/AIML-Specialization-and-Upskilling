# HTTP Verbs: Mastery of PUT, DELETE, and Flask APIs

In the world of web development, creating a **RESTful API** (Representational State Transfer) is a core skill. While **GET** and **POST** handle retrieving and creating data, **PUT** and **DELETE** allow you to modify and remove existing resources.

This guide demonstrates how to build a functional **To-Do List API** using Flask and test it with **Postman**.

---

## 1. Understanding the HTTP Verbs

When building an API, we map specific HTTP methods to standard "CRUD" (Create, Read, Update, Delete) operations.

| HTTP Verb | CRUD Operation | Description |
| --- | --- | --- |
| **GET** | Read | Retrieves a list of items or a specific item by its ID. |
| **POST** | Create | Adds a new item to the collection. |
| **PUT** | Update | Modifies an existing item entirely or partially. |
| **DELETE** | Delete | Removes a specific item from the collection. |

---

## 2. API Data Structure

For this implementation, we simulate a database using a Python list of dictionaries. Each dictionary represents a "Task" in our To-Do list.

```python
from flask import Flask, jsonify, request

app = Flask(__name__)

# Initial Data (Simulated Database)
items = [
    {"id": 1, "name": "Item 1", "description": "This is item 1"},
    {"id": 2, "name": "Item 2", "description": "This is item 2"}
]

```

---

## 3. Implementing the API Routes

### GET: Retrieve All Items

The `jsonify` function converts Python dictionaries or lists into the JSON format required by web clients.

```python
@app.route('/items', methods=['GET'])
def get_items():
    return jsonify(items)

```

### POST: Create a New Task

To add data, we extract the JSON body from the incoming request using `request.json`.

```python
@app.route('/items', methods=['POST'])
def create_item():
    if not request.json or 'name' not in request.json:
        return jsonify({"error": "Item name is required"}), 400
    
    new_item = {
        "id": items[-1]['id'] + 1 if items else 1,
        "name": request.json['name'],
        "description": request.json.get('description', "")
    }
    items.append(new_item)
    return jsonify(new_item), 201

```

### PUT: Update an Existing Task

PUT requests use a **Variable Rule** (the ID) to find the specific resource to change.

```python
@app.route('/items/<int:item_id>', methods=['PUT'])
def update_item(item_id):
    item = next((item for item in items if item['id'] == item_id), None)
    if item is None:
        return jsonify({"error": "Item not found"}), 404
    
    item['name'] = request.json.get('name', item['name'])
    item['description'] = request.json.get('description', item['description'])
    return jsonify(item)

```

### DELETE: Remove a Task

We can remove an item by filtering the list to exclude the specified ID.

```python
@app.route('/items/<int:item_id>', methods=['DELETE'])
def delete_item(item_id):
    global items
    items = [item for item in items if item['id'] != item_id]
    return jsonify({"result": "Item deleted"})

```

## 4. Testing with Postman

Since browsers primarily perform GET requests, developers use **Postman** to test POST, PUT, and DELETE routes.

### How to test your Flask API:

1. **Start the Server:** Run your script (`python app.py`). The server will typically run on `http://127.0.0.1:5000`.
2. **Open Postman:** Create a new request.
3. **Configure the Request:**
* **Method:** Select POST, PUT, or DELETE.
* **URL:** Enter `http://127.0.0.1:5000/items/1` (change the ID as needed).
* **Body:** For POST and PUT, go to the **Body** tab, select **raw**, and choose **JSON**.


4. **Send:** Click the Send button and observe the JSON response from Flask.

---

## 5. Technical Best Practices

* **Status Codes:** Always return appropriate HTTP status codes (e.g., `201` for Created, `404` for Not Found, `400` for Bad Request).
* **Variable Rules:** Use `<int:item_id>` in routes to ensure the ID is automatically converted to an integer before reaching your Python logic.
* **Jsonify:** Use `jsonify()` instead of returning raw dictionaries to ensure the `Content-Type` header is correctly set to `application/json`.
